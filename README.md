# SEO-агент для cyfrovahata.com.ua

Три автоматичні агенти на базі Claude API:

- **`analyst.py`** — щотижня аналізує Search Console + Google Analytics,
  бачить довгострокову історію, шле тактичний звіт і конкретні рекомендації
  в Telegram. **Тільки читає дані, нічого не змінює на сайті.**
- **`executor.py`** — за командою `/do <номер>` готує чернетку/ревізію
  за зразком існуючого дизайну сайту; за командою `/published <номер>`
  фіксує точку відліку для оцінки ефекту. **Завжди лишає публікацію людині.**
- **`monthly_report.py`** — раз на місяць глибокий стратегічний звіт на
  Claude Opus: технічний SEO-аудит, проксі для лідів, пріоритети на місяць.

Усі три запускаються самі за розкладом через GitHub Actions — нічого
не потрібно тримати запущеним на своєму комп'ютері чи окремому сервері.

---

## Крок 1. WordPress: створити доступ для агента

1. У wp-admin створи **окремого користувача** з роллю **Editor** (не Admin!),
   наприклад `seo-agent`. Це обмежує шкоду, якщо ключ колись витече.
2. Зайди під цим користувачем у **Users → Profile → Application Passwords**.
3. Введи назву (наприклад "Claude SEO Agent") → **Add New Application Password**.
4. Скопіюй пароль, що з'явиться (формат `xxxx xxxx xxxx xxxx xxxx xxxx`) —
   знадобиться нижче як `WP_APP_PASSWORD`.
5. Переконайся, що REST API доступний: відкрий
   `https://cyfrovahata.com.ua/wp-json/wp/v2/posts` — має повернутись JSON.

## Крок 2. Google Cloud: доступ до Search Console і GA4

1. Створи проєкт у [Google Cloud Console](https://console.cloud.google.com/).
2. Увімкни два API: **Google Search Console API** та **Google Analytics Data API**.
3. Створи **Service Account** (IAM & Admin → Service Accounts), потім
   **Keys → Add Key → JSON** — завантажиться файл-ключ. Його повний вміст
   піде в секрет `GOOGLE_SERVICE_ACCOUNT_JSON`.
4. Додай email цього service account'у (виглядає як `xxx@yyy.iam.gserviceaccount.com`):
   - у **Search Console** → Settings → Users and permissions → Add user (досить **Restricted**);
   - у **GA4** → Admin → Property Access Management → Add (роль **Viewer**).
5. Знайди `GSC_SITE_URL` (точно як у Search Console, напр. `https://cyfrovahata.com.ua/`)
   та `GA4_PROPERTY_ID` (числовий ID властивості, GA4 Admin → Property Settings).

## Крок 3. Telegram-бот

1. У Telegram напиши **@BotFather** → `/newbot` → отримаєш `TELEGRAM_BOT_TOKEN`.
2. Напиши своєму новому боту будь-яке повідомлення (напр. "привіт").
3. Відкрий у браузері:
   `https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/getUpdates`
   і знайди там `"chat":{"id": ...}` — це твій `TELEGRAM_CHAT_ID`.

## Крок 4. GitHub: розмістити код і додати секрети

1. Створи **приватний** репозиторій на GitHub, заливаєш туди весь цей проєкт.
2. Settings → Secrets and variables → Actions → **New repository secret**,
   додай по одному всі ці значення:

   | Назва секрету | Що це |
   |---|---|
   | `ANTHROPIC_API_KEY` | ключ з [console.anthropic.com](https://console.anthropic.com) |
   | `TELEGRAM_BOT_TOKEN` | з кроку 3 |
   | `TELEGRAM_CHAT_ID` | з кроку 3 |
   | `WP_BASE_URL` | `https://cyfrovahata.com.ua` |
   | `WP_USERNAME` | `seo-agent` (логін з кроку 1) |
   | `WP_APP_PASSWORD` | application password з кроку 1 |
   | `GOOGLE_SERVICE_ACCOUNT_JSON` | повний вміст JSON-файлу з кроку 2 |
   | `GSC_SITE_URL` | з кроку 2 |
   | `GA4_PROPERTY_ID` | з кроку 2 |

3. Перейди в **Actions** → переконайся, що workflows активні.
4. Запусти `SEO Analyst` вручну (кнопка **Run workflow**) — перевір, чи прийшов
   звіт у Telegram. Якщо помилка — дивись логи запуску в Actions, там видно
   на якому кроці й чому впало (зазвичай — неправильний секрет).

## Крок 5 (новий). PageSpeed API — для технічного аудиту

Потрібен ще один безкоштовний ключ для щомісячного технічного аудиту:

1. У тому ж проєкті Google Cloud (з кроку 2) увімкни **PageSpeed Insights API**
2. APIs & Services → Credentials → **Create Credentials → API key**
3. Скопійований ключ додай у GitHub як секрет `PAGESPEED_API_KEY`
   (Settings → Secrets and variables → **Actions** → New repository secret)

Це повністю безкоштовний інструмент з щедрою квотою — для одного сайту раз
на місяць вистачить з великим запасом.

## Три рівні звітів

| Звіт | Частота | Модель | Що робить |
|---|---|---|---|
| **SEO Analyst** (`analyst.py`) | щотижня | Claude Sonnet | тактичні зміни, точкові рекомендації, відстеження ефекту минулих правок |
| **SEO Executor** (`executor.py`) | кожні 4 години | Claude Sonnet | виконує `/do` і `/published` команди з Telegram |
| **SEO Monthly Strategic Report** (`monthly_report.py`) | 1 раз на місяць | **Claude Opus** | глибокий стратегічний синтез: повна історія, технічний аудит, проксі для лідів через події GA4, пріоритети на місяць |

Щомісячний звіт свідомо працює на сильнішій (і дорожчій) моделі — Opus —
бо тут важлива глибина одного разу на місяць, а не швидкість. Тижневі
тактичні звіти лишаються на швидшій Sonnet, щоб не витрачати зайве на
рутинні перевірки.

**Важливо:** в GA4 ще не налаштовані формальні "Key events" (цілі/конверсії).
Щомісячний звіт обходить це, рахуючи всі події напряму — Claude сам визначає
зі списку назв подій, що схоже на лід (форма, дзвінок, заявка). Коли матимеш
час — рекомендую зайти в GA4 → Адмін → Події і позначити потрібні події як
"Ключова подія" (Key event) — це 30 секунд кліками, дасть точніші дані в
майбутньому, але вже зараз система працює і без цього.

## Як це працює день у день

- Кожен тижневий запуск **не замінює** попередній аналіз, а **додається** до
  довгострокової історії (`data/metrics_history.json`). Завдяки цьому агент
  бачить тренди за місяці, а не лише "цей тиждень проти минулого".
- **Щопонеділка о 7:00** прилітає звіт у Telegram з тим, що змінилося,
  і список рекомендацій з номерами (`#1`, `#2`, ...).
- Хочеш, щоб агент підготував зміну за якоюсь рекомендацією — пишеш боту
  `/do 1`. Протягом 4 годин прийде одне з двох:

  **Нова сторінка/стаття** → звичайна **чернетка (draft)** в wp-admin.

  **Правка вже опублікованої сторінки** → **autosave-ревізія**, прив'язана
  до тієї ж сторінки; живий контент не зачіпається, доки ти сам не
  відкриєш редактор і не натиснеш "Оновити".

- **Коли сам опублікував/застосував зміну** в wp-admin — напиши боту
  `/published 1`. Агент зафіксує поточні показники цієї сторінки як
  "точку відліку" (baseline).
- **Через 14+ днів** наступний тижневий звіт сам порівняє показники "до" і
  "після" для цієї сторінки і прямо скаже, чи зміна реально допомогла
  (з конкретними цифрами), без додаткових команд з твого боку.
- **Публікуєш ти сам** в усіх випадках — агент ніколи не публікує і не
  змінює живий контент без твого ручного підтвердження.

## Безпека

- Окремий обмежений користувач WordPress (Editor, не Admin) — на випадок витоку ключа.
- Усі Google-доступи — тільки read-only.
- Жодна зміна на сайті не публікується автоматично — тільки чернетки.
- Рекомендую додатково тримати плагін бекапів (наприклад UpdraftPlus) —
  це не заміна цій системі, а додаткова страховка про всяк випадок.

## Якщо захочеш змінити частоту

- Звіт раз на тиждень — забагато чи замало? Зміни рядок `cron` у
  `.github/workflows/seo-analyst.yml` (формат стандартний cron).
- Перевірку команд `/do` частіше/рідше — так само в `seo-executor.yml`.
